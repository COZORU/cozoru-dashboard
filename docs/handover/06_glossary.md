# 06. 用語集（IRIAM・経営指標）

> 最終更新: 2026-06 ／ 呼び名の対応表と最低限のドメイン知識。数式レベルの定義は [PL_FIELD_DEFINITIONS.md](PL_FIELD_DEFINITIONS.md)。

## 同じものの呼び名違い（最重要・混乱の元）

| このシステム/コード | 経営指標.xlsx（旧手集計）の呼び名 | 意味 |
|---|---|---|
| **ダイヤボーナス** | 投げ銭報酬・**マネジメントフィー（MF）** | iriam→事務所への**新規・移籍ライバー獲得インセンティブ**。原資はプラットフォーム。ライバーの手取りに影響しない |
| **事務所ダイヤ** | **レベシェ30%手数料** | ライバーの取り分から差し引くマネジメント手数料 = 合計ダイヤ×(1−料率/100)。全ライバー対象 |

※「投げ銭報酬」という名前だが、**ライバーから取る分ではない**（取る分はレベシェの方）。最も誤解されやすいポイント。

## 基本用語

| 用語 | 意味 |
|---|---|
| 獲得pt | ファンが送ったポイント（生値）。約6.4pt = 1ダイヤ |
| 応援ダイヤ | 投げ銭由来のダイヤ。**Tier判定はこれのみで行う** |
| 時間ダイヤ | 配信時間由来（時給的）。プラットフォーム原資 |
| 合計ダイヤ | 応援＋時間 |
| ライバーダイヤ | 合計ダイヤ × 料率（ライバー取り分） |
| Tier | T1: 応援ダイヤ≥30,000 / T2: ≥10,000 / T3: それ未満 / T4: 0 |
| 成長判定区分 | 月次の「最高/基本/最低」（会社単位）。Tier係数を補正（最高+40% / 最低−30%）。iriamダッシュボードで確認し M_月次ボーナス に手入力 |
| 配信者種別 | 新規 / 移籍 / 既存。**ダイヤボーナスの対象は新規・移籍のみ**（既存は対象外）。応援ダイヤ系KPIの多くは既存を除外して集計 |

## CPN（キャンペーン報酬）

| 種別 | 内容 | 単価 | CSV列 |
|---|---|---|---|
| C5 | 30日50時間でC5ランク到達 | ¥60,000 | T列 |
| B2 | デビューライバーB2到達応援 | ¥75,000（**〜2025-12発生分で廃止**） | **X列**（W列ではない・要注意） |
| A | A1ランク到達 | ¥40,000 | U列 |
| S | S1ランク到達 | ¥60,000 | V列 |
| その他報酬 | 配信応援・デビューイラスト等 | — | W列（**売上計算に含む**） |

## 売上の式（トップ式）

```
売上（税込・入金額）= （ダイヤボーナス + CPN報酬 + 事務所ダイヤ + その他報酬W）× 1.10
                      ≒ M_月次ボーナス D列の iriam実額（請求書と一致するのが正常）
```

- 比率の目安: ダイヤボーナス ~70% / CPN ~13% / 事務所ダイヤ ~10%
- 各レーベル別売上(税抜) = 投げ銭報酬(MF) + C5 + B2 + A + S + その他報酬 + レベシェ

## バナイベ用語

| 用語 | 意味 |
|---|---|
| バナイベ / トップバナーチャレンジ | 応援pt順のランキングイベント。**毎週火曜開始・月曜終了**、月4回前後、1回100人超参加 |
| 入賞 | **100位以内**（1〜3位はアプリトップバナー掲載、4〜100位も称号・特集ページ） |
| EventId | 回の識別子。**先頭8桁 = 実イベント開始日 = 集計の期間キー** |
| BlockId | 回内のブロック（EventId×BlockId = 1つの独立ランキング）。2026年分は元データで空 |
| 回別 / 月次 | ダッシュボードの2ビュー。回別=直近4回、月次=開始日の属する月で束ねた集計 |

## 集計上の決め事（なぜ数字がそうなるかの根拠）

- 入賞判定は **順位（rank≤100）が正**。元データの入賞フラグは最新週が不完全
- 応援ダイヤ系KPIは**既存ライバーを除外**（新規+移籍+不明のみ）— 2026-05-02 クライアント合意
- 登録ライバー数は**未配信を除外** — 同上
- 売上の正は**請求書（入金額）**。旧・経営指標.xlsx と1〜2月に1%程度の差があるが、本システムが正（クライアント承認済み）
