# 04. 月次運用 Runbook

> 最終更新: 2026-06 ／ 毎月の定常作業。現在の実務担当は uyet 阿部様（引き継ぎ後の体制はMTGで確認）。

## 毎月の作業（約10分・**順序厳守**）

1. **iriam管理画面で当月の成長判定区分を確認**（最高 / 基本 / 最低 — 会社単位・単月）
2. **スプシ「経営指標dashboard」の `M_月次ボーナス` に当月行を入力**
   - 1行 = 事務所（3社分）。列: 対象月 / 事務所名 / 区分 / **D列 = iriam実額（入金額・税込）**
   - ⚠️ **これを飛ばして手順4を実行すると売上が¥0で表示される**（D列未入力 = 0で上書きする仕様）
3. **iriam管理画面から CSV 6枚をダウンロードして Drive `dashboard_input/` フォルダへ**
   - 3事務所 × 2種: `YYYYMMDD_YYYYMMDD_streaming_report_<事務所>.csv` / `YYYYMM_monthly_invoice_report_<事務所>.csv`
4. **スプシメニュー「📊 ダッシュボード」→「📥 ① CSV取込 & ダッシュボード更新」**
   - 押さなくても毎日10時台のトリガーで自動処理される（急ぎのときだけ手動）
5. **確認**
   - PL（個社別）の売上(税込) = iriam請求書の金額と一致（**端数±数十円は端数処理で正常**）
   - `取込ログ` シートにエラーがない
   - https://cozoru-dashboard.vercel.app の各画面に当月が出ている

## D列（iriam実額）をあとから直すとき

- 親セクションの売上は SUMIFS 参照なので**修正は即時反映**（再実行不要）
- 過去月をまとめて直した場合は「💰 ②-2 全月一括 売上再同期」を実行

## バナイベ（月次作業なし）

- OMNIAスプシ → IMPORTRANGE で自動連携。CSV投入も入力も不要
- **進行中の回は数値がライブで増える**（pt・入賞数が日中変わるのは仕様。バグではない）
- 画面に「※イベントID未設定 N名」が出たら banner_active のE列欠損（現状0件。出たらOMNIA側に確認）
