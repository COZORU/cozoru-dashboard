| 項目 | 定義 | 算出方法 | 色 |
|---|---|---|---|
| **売上：cozoru / ライブナウV / Tolance** （税込） | iriam請求書相当の税込売上 | `=税抜売上行 × 1.1` または `ROUND(SUMIFS(...)*1.1,...)` | 🟢/🌿 |
| **税抜売上** | iriam支払い4源泉 + その他 + レベシェ | `SUMIFS(RAW!Y:Y, 月, 事務所) + その他CPN + レベシェ` | 🟢 |

## 2. ダイヤ・pt系

| 項目 | 定義 | 算出方法 | 色 |
|---|---|---|---|
| **総ダイヤ数** | 全ダイヤ合計（応援+時間） | `=応援ダイヤ + 時間ダイヤ` | 🌿 |
| **獲得pt数（合計）** | アプリ上で投げられた金額 1pt=1円 | `SUMIFS(RAW!I:I, 月, 事務所)` | 🟢 |
| **Tier1/2/3 獲得pt** | Tier別の獲得pt | `=応援ダイヤ Tier別 × 6` （1ダイヤ=6pt換算） | 🌿 |
| **応援ダイヤ（既存除外、合計）** | 新規・移籍ライバーの応援ダイヤ合計 | `SUMIFS(RAW!P:P, 月, 事務所, !AB!=既存)` | 🟢 |
| **Tier1/2/3 応援ダイヤ** | Tier別 既存除外応援ダイヤ | `SUMIFS(... !AC=N, !AB!=既存)` | 🟢 |
| **時間ダイヤ** | プラットフォーム原資（時給由来） | `SUMIFS(RAW!O:O, 月, 事務所)` | 🟢 |

**Tier判定**: 応援ダイヤ Tier1 ≥ 30,000 / Tier2 = 10,000-30,000未満 / Tier3 < 10,000

## 3. マネジメントフィー（MF）

| 項目 | 定義 | 算出方法 | 色 |
|---|---|---|---|
| **投げ銭報酬（MF合計）** | Tier別MFの合計 | `=Tier1MF + Tier2MF + Tier3MF` | 🌿 |
| **Tier1/2/3 マネジメントフィー** | 応援ダイヤ × (Tier料率 + 月次ボーナス補正) | `=応援ダイヤTier × (基本MF率 + 月次補正)` | 🌿 |
| **ダイヤボーナス（利率）** | MF ÷ 応援ダイヤ（既存除外） | `=MF合計 ÷ 応援ダイヤ既存除外合計` | 🌿 |

**月次ボーナス**: M_月次ボーナスマスタで `基本 / 最高 / 最低` を月別・事務所別に判定。
- 最高 = +0.40 補正
- 最低 = -0.30 補正

## 6. レベシェ系

| 項目 | 算出方法 | 色 |
|---|---|---|
| **レベシェ30%手数料（合計）** | `SUMIFS(RAW!Y:Y, 月, 事務所)` | 🟢 |
| **応援ダイヤ部分** | `SUMPRODUCT((月)*(事務所)*P*(1-AA/100))` | 🟢 |

## 7. ライバー基盤

| 項目 | 定義 | 算出方法 | 色 |
|---|---|---|---|
| **累計所属ライバー数 / 登録ライバー数** | 過去に1回でも配信したライバー累計 | `COUNTIFS(RAW!G:G,"<>", 月, 事務所)` | 🟢 |
| **累計アクティブライバー数** | 当月配信日数 > 0 | `COUNTIFS(... !AD=TRUE)` | 🟢 |
| **アクティブ率** | アクティブ ÷ 登録 | `=アクティブ ÷ 登録` | 🌿 |
| **デビュー数** | 初回配信日時が当月内のライバー | `COUNTIFS(... !AE=TRUE)` | 🟢 |
| **C5達成率（当月デビュー組×30日以内）** | 当月デビュー組のうち30日以内にC5達成 | `_ライバープロファイル!H列="達成" の人数 ÷ デビュー数` | 🟢 |
| **C5達成数（当月デビュー組）** | 同上の達成人数 | 同上分子 | 🟢 |

## 10. 予測月（2026-05以降）

### DB_ライバー月次 の予測列（col18〜21）

| 項目 | 算出方法 |
|---|---|
| +1/2/3ヶ月予測ダイヤ | `calcForecast(diaHistory)` = 直近3ヶ月平均 × トレンド補正（±50%クリッピング） |
| 予測Tier | 予測ダイヤから再判定（T1≥30,000 / T2≥10,000 / T3<10,000）|
| バナイベ貢献 | 今月デビュー数 × 過去バナイベ月の平均ダイヤ（RAWから自動集計）|

### PL(個社別) の予測月セル（applyForecastFormulas）

| 項目 | 算出方法 |
|---|---|
| デビュー数 | `=AVERAGE(直近3ヶ月)` |
| アクティブ平均金額 | `=AVERAGE(直近3ヶ月)` |
| ダイヤボーナス利率 | `=SUM(直近3ヶ月MF) ÷ SUM(直近3ヶ月応援ダイヤ)` |

**注意**: PLの予測は月次ボーナス区分（最高/基本/最低）を反映しない。区分が変わる月は手入力で上書きすること。

## 12. セクション階層と数式パターン

### cozoru:全社 ⇄ cozoruレーベル ⇄ D3レーベル の関係

```
cozoru:全社 (cozoru小計、defCol=3) = cozoruレーベル + D3レーベル
  ├── cozoruレーベル (sub、defCol=5、AK="株式会社cozoru" フィルタ)
  └── D3レーベル    (sub、defCol=4、AK="D3" フィルタ)
```

### Tolance:全社 ⇄ Tolance + BUBBLE + ... の関係

```
Tolance:全社 (Tolance小計、defCol=7) = Tolance + BUBBLE + Deeper Deeper + ... 等13個
  ├── Tolance              (sub、AK="Tolance")
  ├── BUBBLE               (sub、AK="BUBBLE")
  ├── Deeper Deeper        (sub、AK="Deeper Deeper")
  ├── Mofile               (sub、AK="Mofile")
  ├── ヴィラプロ           (sub、AK="ヴィラプロ")
  └── アライアンス：◯◯ × 7
```

### 上部全社合計 ⇄ 各社の関係

```
上部 row 4-56 (全社合計エリア)
= cozoru:全社 (row 58+offset) + ライブナウV (row 262+offset) + Tolance:全社 (row 328+offset)
```

各上部行 N (4≤N≤56) の数式パターン:
- **単純合算**: `=<col><cozoru:全社 row> + <col><ライブナウV row> + <col><Tolance:全社 row>`
- **派生計算**: row 22 (時間ダイヤ比率) = `=<col>17 / <col>13`、row 56 (アクティブ率) = `=<col>45/<col>44`
- **row 23 (投げ銭平均額)**: `=IFERROR(<col>8/<col>50, 0)` （投げ銭報酬 ÷ アクティブ数）

## 13. 旧PLとの差異（よくある原因）

| 差の理由 | 例 |
|---|---|
| 旧PLの定義が古い/手打ち固定値 | ダイヤボーナス利率 旧=0.6固定 / 新=実計算0.72（成長ボーナス込み）|
| 計算式の解釈違い | 投げ銭平均額：旧PL=応援÷登録 / 新=投げ銭報酬÷アクティブ数 |
| RAWデータが旧PL作成後に更新 | cozoru 登録ライバー数 旧=733 / 新=1005 |
| 旧PLが単位ミス（人数 vs 比率） | A：報酬利率 旧=11(人数) / 新=0.009(比率) |
| 旧PLにcozoruレーベル/D3レーベルの概念無し | 新PLで階層化、サブセクション値0 → 親 cozoru:全社 で集計 |

**これらは新PL（経営指標）が正しい**ケース。
