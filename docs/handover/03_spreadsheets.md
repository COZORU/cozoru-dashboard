## 経営指標dashboard（データ基盤）の主要シート

### データ層
| シート | 内容 |
|---|---|
| `RAW_ライバー月次` | CSV取込先。1行 = ライバー×月。全集計の源泉 |
| `RAW_日次` | 日次CSV（15_DailyIngest） |
| `banner_active` | バナイベ実績（OMNIAから IMPORTRANGE・自動更新） |
| `取込ログ` | CSV処理の実行ログ（トラブル時に最初に見る） |

### マスタ層（M_*）
| シート | 内容 | 更新頻度 |
|---|---|---|
| `M_月次ボーナス` | **毎月手入力**: 事務所・成長判定区分（最高/基本/最低）・**D列=iriam実額（入金額税込）**。D列がPL売上の正になる | 毎月 |
| `M_事務所` | Tier別ベース率・補正率（3社共通: T1=0.80/T2=0.70/T3=0.30、最高+0.40/最低-0.30） | ほぼ固定 |
| `M_Tier` | Tier閾値（T1: 応援ダイヤ≥30,000 / T2: ≥10,000） | 固定 |
| `M_CPN` | CPN単価（C5=¥60,000 / A=¥40,000 / S=¥60,000 / B2=¥75,000は廃止済み） | 固定 |
| `M_税率` | 10% | 固定 |
| `M_換算レート` | 1ダイヤ=1円 | 固定 |

### プレゼン層（DB_* — GASが再構築する出力シート）
DB_サマリ・DB_ライバー月次・DB_新人C5達成率・DB_成長進捗・DB_セグメント・DB_デビュー管理・DB_バナー実績集計 など。**手で編集しない**（次回再構築で消える）。

## 経営指標（メイン帳票）

- シート `PL（個社別）`: **17セクション × 53KPI × 月列**（実績全月＋予測）
  - セクション: 全社合計 / cozoru:全社 / cozoruレーベル / D3レーベル / ライブナウV / Tolance:全社 / Tolanceサブ12個
  - 2026-05-24 に全セクション数式100%カバレッジ達成（手打ち値の残存なし）
- **セル背景色の意味**（触る前に必ず確認）:

| 色 | 意味 |
|---|---|
| 🟢 緑 `#a5d6a7` | **iriam実額**（M_月次ボーナスD列参照 = 請求書ベース確定値） |
| 🟡 黄 `#fff9c4` | 数式（セル参照の四則演算） |
| 🔵 青 `#e3f2fd` | RAW集計（SUMIFS/COUNTIFS で RAW_ライバー月次を直接集計） |
| ⚪ 白 | 手入力欄・見出し |

- 売上の構造: `売上(税抜) = 投げ銭報酬(MF) + C5 + B2 + A + S + その他報酬(W列) + レベシェ`、`売上(税込) = ×1.10`。親セクションの売上(税込)は M_月次ボーナスD列の SUMIFS 参照で、**D列を直せば即時反映**

## banner_active 列仕様（バナイベ元データ）

| 列 | 内容 | 注意 |
|---|---|---|
| C | 個社（事務所名） | |
| D | targetDate | **取込基準日。期間判定に使わない**（最新分がズレることがある） |
| E | EventId | **先頭8桁 = 実イベント開始日（火曜）= 期間キーの正** |
| F | イベント名 | |
| G/H | 開始/終了 | 2026年分は空のことがある |
| I | EventType | |
| L | IRIAM ID | 行の有効判定（空行スキップ）に使用 |
| M | ライバー名 | |
| N | レーベル | |
| O | BlockId | **2026年分は空**。同一EventId内のブロック分離は現状不可（[07](07_handover_checklist.md) の未解決事項） |
| P | 順位 | |
| Q | 応援pt | |
| R | 入賞フラグ | **最新週は不完全。入賞判定は P列 rank≤100 が正**（APIはこちらを使う） |

バナイベの基礎知識:
- IRIAM「トップバナーチャレンジ」= 応援pt順のランキングイベント。**毎週火曜開始・月曜終了**、月4回前後
- **入賞 = 100位以内**（1〜3位はトップバナー掲載、4〜100位も称号・特集）
- **EventId × BlockId が1つの独立ランキング**（会社別ではない。全社が同じ回で競う）
